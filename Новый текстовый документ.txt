Тема:Проектирование и разработка электронного учебника «Информационные системы и технологии»

Общая концепция системы
Электронный учебник представляет собой веб-приложение с разделением на клиентскую (React) и серверную (FastAPI) части. Пользователи работают через браузер, все данные (контент, прогресс, результаты тестов, файлы лабораторных) хранятся на сервере. Система поддерживает три роли: студент, преподаватель, администратор. Доступ к функциям определяется через JWT-токены, которые выдаются при входе.

2. Функционал для студентов
2.1. Изучение теоретического материала
Учебник разбит на модули (крупные разделы), внутри каждого модуля — темы(упорядоченный список уроков).
Каждая тема содержит: заголовок, текст с форматированием (заголовки, списки, таблицы, выделение кода), встроенные изображения и схемы, ссылки на видео (YouTube или загруженные файлы), файлы для скачивания (презентации, PDF).
Студент видит оглавление в виде дерева (слева или сверху), может переключаться между темами, система запоминает последнюю открытую тему.
Для сложных терминов реализован интерактивный глоссарий — при клике на термин всплывает определение, также есть отдельная страница со списком всех терминов.
2.2. Контроль знаний (тестирование)
После каждой темы студенту предлагается пройти тест (набор вопросов). Тест может иметь ограничение по времени (опционально).
Типы вопросов:
Одиночный выбор (один правильный ответ из нескольких)
Множественный выбор (несколько правильных ответов)
Сопоставление (пары «термин — определение»)
Краткий текстовый ответ (проверяется по ключевым словам или точному совпадению)
После отправки ответов система автоматически проверяет тест, показывает:
итоговый балл (в процентах)
какие вопросы отвечены правильно, а какие нет
правильные ответы на ошибочные вопросы
ссылку на соответствующий раздел учебника для повторения материала
Тест можно проходить неограниченное количество раз, но в историю сохраняется лучшая попытка. Чтобы перейти к следующей теме, обычно требуется набрать проходной балл (например, 70%).
2.3. Практические и лабораторные работы
К некоторым темам прикрепляется лабораторная работа: задание с описанием, требования к оформлению, критерии оценки.
Студент может загрузить файл (архив с кодом, отчет в PDF/DOCX, скриншоты) через форму на странице темы.
После загрузки работа получает статус «на проверке». Студент видит свои отправленные работы в личном кабинете.
Когда преподаватель выставляет оценку (баллы и комментарий), студент получает уведомление (в интерфейсе и, опционально, на email). Работа переходит в статус «проверена».
Если оценка неудовлетворительная, студент может загрузить исправленную версию (новая попытка).
2.4. Отслеживание прогресса
На главной странице учебника и в личном кабинете отображается прогресс-бар(процент изученных тем).
Для каждой темы в оглавлении стоит иконка/цвет:
серый — тема не начата
желтый — тема в процессе (тест пройден, но не сдан)
зеленый — тема полностью завершена (тест сдан, лаба проверена, если требуется)
Доступен отдельный экран «Моя статистика» с графиками:
динамика прохождения тем по дням
средний балл за тесты по модулям
количество успешно сданных тестов из всех попыток
Можно посмотреть историю всех попыток тестов (дата, балл, затраченное время).
2.5. Личный кабинет
Просмотр и редактирование профиля (ФИО, email, пароль, аватар)
Список загруженных лабораторных работ с фильтром по статусу (ожидают проверки/проверены)
Настройка уведомлений (на email или только в приложении)
Просмотр финального сертификата (если пройдены все темы) — генерируется в PDF с именем студента, датой и процентом выполнения.

3. Функционал для преподавателя
3.1. Управление контентом (CRUD)
Создание, редактирование, удаление модулей и тем.
Редактор текста тем (WYSIWYG или Markdown с предпросмотром) — позволяет вставлять изображения, видео, таблицы, код.
Управление тестами: добавление/удаление вопросов, изменение вариантов ответов, настройка проходного балла.
Управление лабораторными работами: создание задания, указание максимального балла, прикрепление файлов-образцов.
3.2.Проверка лабораторных работ
Специальная страница «Работы на проверку» со списком всех загрузок студентов (можно фильтровать по группе, теме, дате).
При открытии работы преподаватель видит: задание, загруженный файл (с возможностью скачать/просмотреть online), данные студента.
Форма выставления оценки: поле для балла (целое число), текстовый комментарий (можно шаблонные фразы «нет описания», «ошибка в коде» и т.д.).
После сохранения студент мгновенно видит оценку и комментарий в своем кабинете.
3.3. Мониторинг успеваемости
Дашборд преподавателя с обобщенной статистикой по группе/потоку:
средний процент выполнения по всем студентам
список студентов, отстающих (менее 50% тем)
гистограмма по оценкам за лабораторные работы
Возможность экспортировать отчет (CSV или Excel) по выбранной группе: список студентов, количество пройденных тем, средний балл за тесты, оценки за лабы.
Просмотр детального прогресса конкретного студента (какие темы пройдены, какие тесты не сданы, какие лабы не загружены).
3.4. Управление пользователями
Добавление новых студентов (или импорт из Excel-файла с email и ФИО).
Назначение студентов в группы (для удобного фильтрации).
Сброс пароля студентам (генерация временного пароля, отправка на email).
Блокировка/разблокировка доступа к учебнику.

4. Функционал администратора
Администратор имеет все права преподавателя плюс дополнительные функции:
Управление ролями пользователей (назначить преподавателя, снять права).
Просмотр логов системы (кто когда заходил, какие действия выполнял).
Общие настройки приложения (название курса, контактный email, настройки email-уведомлений, таймаут сессии).
Резервное копирование базы данных и контента (кнопка «Создать бэкап»).
Очистка кэша (Redis) и старых файлов загрузок.

5. Функционал бэкенда (FastAPI) в терминах логики
5.1. Аутентификация и безопасность
Регистрация новых пользователей с подтверждением email (на почту приходит ссылка для активации).
Вход по email и паролю — сервер генерирует пару токенов (access и refresh). Access-токен живет недолго (15–30 минут), refresh-токен хранится в базе и обновляет access-токен автоматически.
Защита эндпоинтов: при запросе к любому API проверяется наличие токена и его валидность, а также роль пользователя (студент не может зайти в панель преподавателя).
Все пароли хранятся в хешированном виде (bcrypt).
5.2. Работа с контентом
Сервер отдает учебный материал в формате JSON (структура модулей и тем). Текст контента хранится в базе как Markdown или HTML.
При запросе темы сервер также возвращает состояние прогресса для текущего пользователя (пройдена/нет, лучший балл за тест).
Для ускорения загрузки часто запрашиваемые данные (оглавление, глоссарий) кэшируются в Redis на несколько часов.
5.3. Обработка тестов
Когда студент отправляет ответы, сервер загружает из базы эталонные ответы и вычисляет балл по правилам:
одиночный выбор: 1 балл за правильный, 0 за неправильный
множественный выбор: частичные баллы (например, за каждый правильно отмеченный вариант)
текстовый ответ: сравнение с несколькими допустимыми вариантами (регистронезависимо, с учетом синонимов)
Результат сохраняется в таблицу попыток (связь студент → тест → балл).
Если набрано больше проходного балла, тема автоматически отмечается как пройденная (но только если все обязательные элементы темы выполнены).
5.4. Файловые операции
Сервер принимает файлы от студентов (лабораторные), переименовывает их по шаблону «ID_студента_ID_лабы_дата.расширение» и сохраняет в защищенную папку (доступную только серверу).
При скачивании файла преподавателем сервер проверяет права (преподаватель может скачать любой файл своей группы, студент — только свои).
Периодически (например, раз в неделю) запускается фоновая задача, удаляющая старые временные файлы и неактивированные аккаунты.
5.5. Генерация отчетов и сертификатов
По запросу преподавателя сервер собирает данные по группе, формирует Excel-файл в фоновом режиме (Celery) и отдает ссылку для скачивания.
При завершении всех тем сервер генерирует PDF-сертификат: вставляет имя студента, дату, QR-код со ссылкой на проверку подлинности.

6.Функционал фронтенда (React) в терминах UI/UX
6.1. Общие интерфейсные решения
Адаптивный дизайн — учебник удобно смотреть как на компьютере, так и на планшете/телефоне (меню сворачивается в гамбургер).
Темная/светлая тема — пользователь может переключать (сохраняется в localStorage).
Умная навигация — хлебные крошки (Модуль 1 → Тема 3 → Тест), боковое меню с быстрым переходом между темами.
Индикация загрузки — скелетоны (skeleton screens) вместо белого экрана пока данные подгружаются.
6.2. Страница изучения темы
Основная область отображает контент темы (с поддержкой подсветки синтаксиса для кода).
Рядом с текстом плавает мини-меню «Быстрый переход» (к тесту, к лабораторной, к глоссарию).
Внизу страницы — кнопки «Предыдущая тема» / «Следующая тема» (если тест сдан).
Боковая панель с прогрессом по текущей теме (прочитано X% контента — отслеживается скроллинг).
6.3. Интерфейс тестирования
Тест открывается либо как отдельная страница, либо в модальном окне на странице темы.
На таймере отображается оставшееся время (если ограничено), при истечении времени тест автоматически отправляется.
Прогресс-бар показывает, сколько вопросов отвечено.
Подсветка сложных вопросов — можно отметить флажком «нужен ответ позже» (чтобы вернуться).
После отправки результаты показываются анимированно: зеленые галочки для правильных ответов, красные крестики для ошибок, с пояснениями.
6.4. Личный кабинет студента
Дашборд — приветствие, круговой график общего прогресса, последняя активность, уведомления (например, «Преподаватель проверил лабораторную работу №3»).
Вкладка «Мой прогресс» — детализированная таблица по всем темам с баллами, датами прохождения.
Вкладка «Лабораторные работы» — карточки каждой сданной работы с текущим статусом (ожидает/проверена), оценкой и комментарием.
Вкладка «Настройки» — смена пароля, настройка уведомлений, удаление аккаунта (с подтверждением).
6.5. Интерфейс преподавателя
Основная навигация: переключение между разделами «Контент», «Проверка работ», «Статистика», «Пользователи».
Редактор контента — форма с полями для заголовка, текста (визуальный редактор), загрузки файлов. Можно менять порядок тем перетаскиванием (drag-and-drop).
Страница проверки лабораторных — список загрузок с предпросмотром: клик по строке открывает панель с файлом и формой оценки. Реализована «быстрая оценка» — стрелки вверх/вниз, чтобы не открывать каждую отдельно.
Статистика — графики на библиотеке Chart.js, интерактивные фильтры (выбрать группу, диапазон дат). Есть кнопка «Скачать отчет» (сервер генерирует Excel).
6.6. Общая навигация и маршрутизация
Маршруты защищены: неавторизованный пользователь видит только страницу входа/регистрации.
Если у пользователя нет прав на раздел (например, студент зашел на /teacher), его перенаправляет на главную страницу учебника с уведомлением.
При истечении access-токена фронтенд автоматически пытается обновить его через refresh-токен (без потери данных формы). Если не получается — отправляет на логин.

7. Дополнительная функциональность (для усиления диплома)
7.1. Чат/форум по каждой теме
Под каждой темой есть блок «Обсуждение» — студенты могут задавать вопросы, преподаватель отвечать.

Производительность
Страница со списком модулей и тем должна загружаться не более 1 секунды (при нормальном интернете).
Одновременно систему должны комфортно использовать до 100 пользователей (студентов и преподавателей) без деградации скорости.
8.2. Безопасность
Реализована защита от CSRF-атак (токен в заголовках).
CORS настроен так, что только доверенный фронтенд-домен может обращаться к API.
Все вводимые пользователем данные экранируются перед отображением (защита от XSS).
Логирование важных действий (вход, изменение контента, выставление оценок) с указанием времени и IP.
8.3. Надежность
Регулярное автоматическое резервное копирование базы данных (раз в сутки).
При ошибке сервера пользователь видит читаемое сообщение (не стектрейс).
Сессии не теряются при обновлении страницы (refresh-токен хранится в httpOnly cookie).
